Oracle Cloud Always Freeの使い方【ARM 24GBを無料で】

2026年3月1日

著者：AI Work Stage 編集部（AI執筆）

Oracle Cloud Infrastructure（OCI）には「Always Free」という永続無料枠があり、ARM（Ampere A1）インスタンスを最大4コア・24GB RAMまで無料で使えます。この記事では、Oracle公式ドキュメントをもとにセットアップ手順を解説します。

Oracle Cloud Always Freeの概要

Oracle Cloud Always Freeは、期限なしで利用できる無料リソースです。2026年3月時点の主なAlways Freeリソースは以下の通りです（公式ドキュメントより）：

  • ARM（Ampere A1）インスタンス：合計4 OCPUs・24GB RAM（1インスタンスでも複数でも可）
  • ストレージ：ブロックボリューム合計200GB
  • ネットワーク：帯域幅10Mbps・月10TB送信
  • オブジェクトストレージ：20GB

※Always Free枠はOracleのポリシーにより変更される可能性があります。必ずOracle公式サイトでご確認ください。

アカウント登録

Oracle Cloud無料アカウントの作成手順：

  1. Oracle Cloud Free Tierにアクセス
  2. 「Start for free」をクリック
  3. メールアドレス・氏名・国などを入力（クレジットカードの登録が必要ですが、Always Free枠内では請求されません）
  4. ホームリージョンを選択（後から変更不可。日本なら「Japan East（Tokyo）」または「Japan West（Osaka）」）
  5. SMS認証・身分確認を完了

⚠️ 注意：ホームリージョンは一度設定すると変更できません。ARMインスタンスは特定リージョンで在庫が逼迫することがあるため、注意が必要です。

ARM仮想マシンの作成手順

ステップ1：コンソールにログイン

OCIコンソール（cloud.oracle.com）にログインし、左上のメニューから「コンピュート」→「インスタンス」に移動します。

ステップ2：インスタンスの作成

  1. 「インスタンスの作成」をクリック
  2. 名前を入力（例：my-arm-server）
  3. 「イメージとシェイプ」セクションで「シェイプの変更」をクリック
  4. 「Ampere」タブを選択し、「VM.Standard.A1.Flex」を選択
  5. OCPUs：4、メモリ：24GBに設定（Always Free上限）
  6. イメージ：Ubuntu 22.04またはOracle Linux（好みで選択）

ステップ3：SSHキーの設定

  1. 「SSHキーの追加」セクションで「SSHキーペアの生成」を選択
  2. 秘密キーをダウンロード（.keyファイル）
  3. または、既存の公開鍵を貼り付けることも可能

ステップ4：インスタンスの起動

「作成」ボタンをクリックするとインスタンスが起動します。パブリックIPアドレスが割り当てられます（数分かかります）。

SSHでの接続

# キーファイルの権限を変更
chmod 600 ~/Downloads/your-key.key

# SSH接続（Ubuntuの場合）
ssh -i ~/Downloads/your-key.key ubuntu@YOUR_PUBLIC_IP

# Oracle Linuxの場合
ssh -i ~/Downloads/your-key.key opc@YOUR_PUBLIC_IP

ファイアウォール設定

OCIのセキュリティリストでポートを開放する必要があります：

  1. コンソールで「ネットワーキング」→「仮想クラウド・ネットワーク」→「セキュリティ・リスト」
  2. 「イングレス・ルールの追加」でポートを開放（例：80, 443, 22）

また、OS側のファイアウォール（iptablesまたはfirewalld）の設定も必要です：

# Ubuntuの場合（ufwを使用）
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw enable

活用例

  • Webサーバー（Nginx・Apache）のホスティング
  • VPNサーバー（WireGuard）の構築
  • Bot・APIサーバーの運用
  • GitHub Actionsのセルフホストランナー
  • Claude APIを使った自動化ツールのホスティング

注意点

  • 在庫不足でARMインスタンスが作成できない場合があります（Out of capacity）
  • 長期間ログインしないとアカウントが非アクティブ化される場合があります
  • Always Free枠を超えると課金が発生するため、使用量のモニタリングを推奨します
