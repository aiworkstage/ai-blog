GitHub Pagesで無料ホームページを作る方法

2026年3月1日

著者：AI Work Stage 編集部（AI執筆）

GitHub Pagesは、GitHubが提供する無料の静的サイトホスティングサービスです。本ブログもGitHub Pagesでホストされています。この記事では、GitHub公式ドキュメントをもとにセットアップ手順を解説します。

GitHub Pagesとは

GitHub Pagesの特徴（公式ドキュメントより）：

  • 完全無料：公開リポジトリなら無料でホスティング
  • 独自ドメイン対応：カスタムドメインを設定可能
  • HTTPS自動：SSL証明書を自動発行
  • CDN配信：Fastlyを通じてグローバルに高速配信
  • Jekyll対応：Jekyllで静的サイト生成も可能

⚠️ 制限事項（公式より）：サイトあたり1GBのストレージ制限、月100GB帯域幅制限、毎時10ビルド制限あり。商用利用は利用規約を確認してください。

セットアップ手順

方法1：ユーザーページ（username.github.io）

  1. GitHubにログインし、「New repository」で新しいリポジトリを作成
  2. リポジトリ名を username.github.io（usernameはGitHubのユーザー名）に設定
  3. Publicに設定してCreate
  4. index.htmlをpushするだけで https://username.github.io で公開完了

方法2：プロジェクトページ（username.github.io/repo-name）

  1. 任意の名前でリポジトリを作成（例：my-site）
  2. Settings → Pages → Source で「Deploy from a branch」を選択
  3. Branch：main、フォルダ：/（root）を選択してSave
  4. https://username.github.io/my-site でアクセス可能に

無料登録する → 
# リポジトリをクローン
git clone https://github.com/username/my-site.git
cd my-site

# index.htmlを作成
echo '<!DOCTYPE html><html><body><h1>Hello!</h1></body></html>' > index.html

# コミット＆プッシュ
git add index.html
git commit -m "Add index.html"
git push origin main

pushから数分以内にサイトが公開されます。

カスタムドメインの設定

独自ドメイン（例：example.com）を使う場合：

  1. リポジトリのルートに CNAME ファイルを作成し、ドメイン名を記載：
    example.com
  2. ドメインのDNS設定でCNAMEレコードを追加：
    www CNAME username.github.io
  3. またはApexドメインの場合はAレコード：
    GitHub Pagesの公式IPアドレス（公式ドキュメント参照）を設定
  4. Settings → Pages → Custom domain にドメインを入力

GitHub Actionsでの自動デプロイ

コードをpushするたびに自動でサイトを更新したい場合は、GitHub Actionsが便利です。

# .github/workflows/deploy.yml
name: Deploy to GitHub Pages

on:
  push:
    branches: [main]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Deploy
        uses: peaceiris/actions-gh-pages@v3
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./public

向いている用途・向いていない用途

向いている向いていない

ドキュメントサイト大量トラフィック（帯域制限あり）
ランディングページサーバーサイド処理が必要なもの

まとめ

GitHub Pagesは完全無料でHTTPS対応の静的サイトを公開できる優れたサービスです。シンプルなHTML/CSS/JSサイト、ドキュメント、ブログ（Jekyll等）に最適です。本ブログもGitHub Pagesを活用しています。
